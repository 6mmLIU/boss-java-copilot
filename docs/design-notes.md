# 前端设计与交互说明

## 产品结构

这个工具是工作台，不是 landing page。首页直接承载实际流程：

1. 配置目标、城市、关键词、薪资和模式。
2. 预检浏览器登录态。
3. 运行复审或自动投递。
4. 查看候选池和日志。
5. 调整筛选规则和跨平台环境。

## 按钮布局

运行页按钮按风险和频率排列：

- 保存配置：低风险、常用，放在左侧。
- 预检登录：运行前置条件，放在保存之后。
- 开始复审：推荐主路径，使用 primary。
- 自动投递：高风险动作，使用 danger-soft，并打开 HeroUI Modal 二次确认。
- 停止：只在任务运行时可用，空闲时禁用，避免误以为可以停止不存在的任务。

## HeroUI 组件使用

项目使用 `@heroui/react` 和 `@heroui/styles`，不复制或修改 HeroUI 源码。组件职责：

- Button：命令动作和状态按钮。
- Tabs：页面分区和键盘可访问导航。
- Card：信息区块和配置区块。
- Chip：状态、计数、城市和候选状态。
- Modal：自动投递二次确认。
- Drawer：候选详情。
- Input/TextArea/Switch：配置和手动评估。
- Table：候选池数据扫描。

## 动效取舍

HeroUI 自带转场用于 overlay 和组件内部状态：

- Modal 打开/关闭。
- Drawer 进入/退出。
- Tabs 的可访问选择状态。

自定义 CSS 动效只用于工作流反馈：

- KPI 数字更新：number pop-in，强调状态变化。
- Tab panel 进入：轻微 reveal，帮助用户感知区域变化。
- 实时日志新行：短距离 row-in，提示新事件。
- 进度条：width transition，表达投递进度。

所有自定义动效都保留 `prefers-reduced-motion` 降级。

## 视觉原则

- 管理工具优先信息密度和可扫描性。
- 卡片圆角控制在 8px。
- 避免纯紫/深蓝 AI dashboard 风格。
- 主色使用青色、蓝色、绿色和少量红色语义色。
- 危险动作不和普通动作混在同一种视觉权重里。
